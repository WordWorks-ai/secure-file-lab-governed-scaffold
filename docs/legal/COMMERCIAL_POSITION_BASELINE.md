# Commercial Position Baseline (2026-03-03)

## Purpose

Define a single current pricing baseline and meeting script that can be used consistently in client conversations.

This document is a commercial planning artifact and not legal, tax, or accounting advice.

## Scope Truth (Always State First)

Current repository state is a hardened **Phase 0/1 scaffold**, not a feature-complete secure file-sharing runtime.

Implemented now:
- deterministic platform bootstrap and reproducibility controls
- hardened container/runtime defaults
- backup and restore smoke validation
- governance/security documentation and test guardrails

Not implemented yet:
- auth/session runtime flow
- file ingest/download runtime pipeline
- full encryption/scan/share/audit runtime coverage

## Offer Architecture

Use a two-product framing:

1. **Product A: Snapshot Transfer**
   - commit-bound repository snapshot rights
   - no implied future roadmap delivery
2. **Product B: Authored Delivery**
   - expedited, direct author involvement for scoped buildout
   - priced as premium delivery for speed, judgment, and responsibility

## Baseline Pitch Numbers

### 1) Snapshot transfer (commit-bound)

- pitch anchor: **$59,000**
- likely close band: **$45,000 to $55,000**
- walk-away floor: **$40,000**

### 2) License + stewardship (control retained)

- upfront license: **$30,000**
- stewardship retainer: **$3,500/month**
- minimum term for baseline economics: **6 months**
- optional later buyout can be offered separately

### 3) Emergency authored build (scoped 4-6 weeks)

- pitch anchor: **$85,000**
- likely close band: **$65,000 to $75,000**
- walk-away floor: **$60,000**

## Final Value Prompt (Use Verbatim)

"Here are the two products.
Product A is the current commit-bound scaffold snapshot, with full scope truth and transfer terms.
Product B is expedited authored delivery where I prioritize your roadmap and deliver under compressed timelines.
If you want only the snapshot, that's one price. If you want speed, judgment, and direct authored execution, that's a premium price."

## Why These Numbers

1. Replacement-cost anchor:
   - internal estimate remains **340 to 560 hours**
   - buyer-side external build cost is typically much higher than low-end snapshot asks
2. Maturity discount:
   - core runtime product paths remain unfinished, so snapshot pricing is discounted below full-product levels
3. Premium only when warranted:
   - urgency and direct authored involvement justify a separate premium track
4. Opportunity-cost protection:
   - floors are set so external work must beat continuing internal roadmap development

## Negotiation Guardrails

1. Keep snapshot value and authored-delivery value separate.
2. Never bundle snapshot rights, emergency execution, and ongoing support into one undifferentiated number.
3. Keep scope truth explicit in every quote and agreement.
4. Bind rights to a specific commit hash and explicit include/exclude schedule.
5. If economics fall below floor with high support expectations, decline.

## Update Triggers

Re-baseline numbers when one of these changes:
- a complete end-to-end runtime slice is implemented (auth + file + scan/audit path)
- buyer requests broad exclusivity or strategic rights beyond snapshot transfer
- support burden or delivery timeline materially changes
