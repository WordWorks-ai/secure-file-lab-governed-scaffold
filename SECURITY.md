# Security Policy

## Reporting A Vulnerability

If you believe you have found a security vulnerability in this repository, do
not open a public issue.

Report it privately to `scott.caesar@gmail.com` with:

- a clear description of the issue
- affected components or file paths
- reproduction steps or proof-of-concept details
- impact assessment if known

I will review reports as quickly as possible and coordinate remediation before
public disclosure when appropriate.

## Scope

This repository is a governed prototype and portfolio artifact. Reports are
most useful when they focus on:

- authentication and session handling
- encryption and key-management flows
- file lifecycle and malware-gating behavior
- audit-chain integrity
- backup, restore, and secret-rotation workflows

## Disclosure Expectations

Please allow reasonable time for triage and remediation before sharing details
publicly.
